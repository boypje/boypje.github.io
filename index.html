AI AGENT HOSPITAL

AI Agent for Hospital Management

Intelligent Decision Support & Data Exploration

An AI-powered assistant ecosystem designed to help hospital staff explore operational data and internal documents through natural language queries.

  • Dual AI Agent Architecture: Consists of two specialized agents — a RAG-based document agent using hospital PDF documents as its knowledge source, and a database agent built on vectorized hospital data.
  • Document RAG Agent: Hospital policies, SOPs, and reports are ingested from Google Drive, processed into embeddings, and queried contextually to deliver accurate, source-aware answers.
  • Database Vector Agent: Operational data from PostgreSQL (Supabase) is vectorized, enabling semantic search and analytical Q&A over structured hospital data.
  • Orchestration & Automation: n8n is used to automate data ingestion, syncing from Google Drive, database updates, and agent execution pipelines.
  • Intelligent Query Routing: Python-based LangChain acts as the reasoning layer, dynamically classifying user queries and routing them to the appropriate agent and AI model based on question type and priority.

Tech Stack

Python LangChain n8n PostgreSQL Supabase Vector Database Google Drive API
